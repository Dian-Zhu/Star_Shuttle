## 2. 架构概述

### 2.1 架构设计原则

1. **安全性优先**：所有设计决策都优先考虑系统安全性，特别是凭证管理和数据传输安全
2. **跨平台兼容性**：确保在 Windows、macOS 和 Linux 平台上具有一致的功能和用户体验
3. **高性能**：优化终端响应速度和文件传输效率，确保流畅的用户体验
4. **模块化设计**：采用松耦合的模块化架构，提高代码可维护性和可扩展性
5. **分层架构**：清晰分离关注点，便于独立开发和测试
6. **可测试性**：设计易于测试的组件和接口

### 2.2 整体架构模式

系统采用**分层架构模式**，结合 Tauri 2 的主进程-渲染进程模型，主要分为以下三层：

1. **前端层**（渲染进程）：负责用户界面渲染和用户交互
2. **中间层**（IPC 通信）：负责前端和后端之间的通信
3. **后端层**（主进程）：负责核心功能实现，包括 SSH 连接、终端仿真、文件传输等

### 2.3 Tauri 2 应用架构

Tauri 2 应用采用双进程模型：

- **主进程**：使用 Rust 编写，运行在系统原生环境中，具有完全的系统访问权限，负责处理敏感操作和核心业务逻辑
- **渲染进程**：使用 Web 技术（HTML/CSS/JavaScript）编写，运行在安全的沙箱环境中，负责用户界面渲染和用户交互

两个进程之间通过 Tauri 提供的 IPC 机制进行通信，包括命令调用和事件传递。

### 3.2 前端组件

#### 3.2.1 连接管理界面

- **功能**：管理 SSH 连接配置，包括创建、编辑、删除、组织连接
- **组件结构**：
  - 连接列表组件
  - 连接详情组件
  - 连接编辑表单组件
  - 连接分组组件
- **技术实现**：使用响应式 Web 框架（如 Svelte）构建，支持拖拽操作和实时搜索

#### 3.2.2 终端模拟器 UI

- **功能**：提供远程终端的可视化界面
- **组件结构**：
  - 终端显示组件
  - 终端控制工具栏
  - 会话标签组件
- **技术实现**：使用 Web 技术模拟终端界面，支持 ANSI 颜色和终端特性

#### 3.2.3 文件传输界面

- **功能**：提供本地与远程文件系统的交互界面
- **组件结构**：
  - 双面板文件浏览器
  - 文件操作工具栏
  - 传输队列组件
  - 传输进度组件
- **技术实现**：支持拖拽操作，实时显示文件系统变化

#### 3.2.4 设置面板

- **功能**：管理应用配置和用户偏好
- **组件结构**：
  - 通用设置组件
  - 终端设置组件
  - 文件传输设置组件
  - 安全设置组件
- **技术实现**：表单驱动的配置管理，支持实时预览

### 3.3 后端模块

#### 3.3.1 SSH 连接管理器

- **功能**：管理 SSH 连接的生命周期，包括连接建立、维护和关闭
- **核心职责**：
  - 连接配置验证和解析
  - 多种认证方式支持（密码、私钥、代理转发）
  - 连接池管理
  - 连接状态监控
- **技术实现**：基于 russh 库实现 SSH 协议支持

#### 3.3.2 终端仿真引擎

- **功能**：处理终端输入输出，模拟终端行为
- **核心职责**：
  - 终端输入处理和输出渲染
  - ANSI/VT100 转义序列解析
  - 终端窗口大小调整处理
  - 滚动缓冲区管理
- **技术实现**：结合 russh 的终端支持和自定义终端仿真逻辑

#### 3.3.3 SFTP/SCP 文件传输引擎

- **功能**：处理本地与远程系统之间的文件传输
- **核心职责**：
  - SFTP/SCP 协议实现
  - 文件上传和下载
  - 目录操作（创建、删除、重命名）
  - 文件权限管理
  - 传输队列管理
- **技术实现**：基于 russh-sftp 库实现 SFTP 支持，自定义 SCP 实现

#### 3.3.4 凭证安全存储模块

- **功能**：安全存储和管理 SSH 凭证信息
- **核心职责**：
  - 密码加密存储
  - 私钥安全管理
  - 平台原生安全存储集成
  - 凭证访问控制
- **技术实现**：使用 keyring 库集成平台原生安全存储（Windows 凭证管理器、macOS Keychain、Linux Secret Service API）

#### 3.3.5 会话管理模块

- **功能**：管理用户会话状态
- **核心职责**：
  - 会话状态保存和恢复
  - 多会话并发管理
  - 会话配置管理
- **技术实现**：基于 SQLite 数据库实现会话持久化

#### 3.3.6 日志记录模块

- **功能**：记录系统运行日志
- **核心职责**：
  - 连接历史记录
  - 终端会话日志
  - 文件传输日志
  - 错误和事件日志
- **技术实现**：使用 Rust log 库实现，支持不同日志级别和输出目标

#### 3.3.7 配置管理模块

- **功能**：管理应用配置和用户偏好
- **核心职责**：
  - 配置文件读写
  - 配置验证
  - 配置变更通知
- **技术实现**：基于 TOML 格式的配置文件，支持实时配置更新

#### 3.3.8 错误处理模块

- **功能**：统一处理系统错误
- **核心职责**：
  - 错误捕获和分类
  - 错误信息格式化
  - 错误日志记录
  - 用户友好的错误提示
- **技术实现**：使用 Rust 自定义错误类型，结合日志记录

## 4. 模块交互与通信

### 4.1 前端-后端 IPC 机制

系统使用 Tauri 提供的 IPC 机制实现前端和后端之间的通信，主要包括两种方式：

1. **命令调用**：前端主动调用后端提供的命令，获取返回结果
2. **事件传递**：后端主动向前端发送事件，通知状态变化或异步操作结果

### 4.3 关键流程交互时序图

#### 4.3.1 SSH 连接建立流程

```
┌───────────┐     ┌───────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 前端 UI   │────▶│ IPC 层    │────▶│ SSH 连接管理器  │────▶│ 凭证安全存储     │
└───────────┘     └───────────┘     └─────────────────┘     └────────┬────────┘
                                                                     │
┌───────────┐     ┌───────────┐     ┌─────────────────┐     ┌────────▼────────┐
│ 前端 UI   │◀────│ IPC 层    │◀────│ SSH 连接管理器  │◀────│ 远程 SSH 服务器 │
└───────────┘     └───────────┘     └─────────────────┘     └─────────────────┘
```

1. 用户在前端 UI 选择连接并点击连接按钮
2. 前端通过 IPC 调用 SSH 连接管理器的连接方法
3. SSH 连接管理器从凭证安全存储获取必要的凭证信息
4. SSH 连接管理器与远程 SSH 服务器建立连接
5. 连接成功后，SSH 连接管理器通过 IPC 向前端返回连接状态
6. 前端更新 UI，显示连接成功信息

#### 4.3.2 终端数据传输流程

```
┌───────────┐     ┌───────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 终端 UI   │────▶│ IPC 层    │────▶│ 终端仿真引擎     │────▶│ SSH 连接管理器  │
└───────────┘     └───────────┘     └─────────────────┘     └────────┬────────┘
                                                                     │
┌───────────┐     ┌───────────┐     ┌─────────────────┐     ┌────────▼────────┐
│ 终端 UI   │◀────│ IPC 层    │◀────│ 终端仿真引擎     │◀────│ 远程 SSH 服务器 │
└───────────┘     └───────────┘     └─────────────────┘     └─────────────────┘
```

1. 用户在终端 UI 输入命令
2. 前端通过 IPC 将输入数据发送到终端仿真引擎
3. 终端仿真引擎处理输入数据，并通过 SSH 连接管理器发送到远程服务器
4. 远程服务器执行命令，返回输出结果
5. SSH 连接管理器将输出结果传递给终端仿真引擎
6. 终端仿真引擎处理输出结果，包括解析 ANSI 转义序列
7. 终端仿真引擎通过 IPC 将处理后的输出数据发送到前端
8. 前端更新终端 UI，显示输出结果

#### 4.3.3 文件传输流程

```
┌───────────┐     ┌───────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 文件传输 UI│────▶│ IPC 层    │────▶│ SFTP/SCP 引擎    │────▶│ SSH 连接管理器  │

└───────────┘     └───────────┘     └─────────────────┘     └─────────────────┘
```

1. 用户在文件传输 UI 选择文件并点击传输按钮
2. 前端通过 IPC 将传输请求发送到 SFTP/SCP 引擎
3. SFTP/SCP 引擎通过 SSH 连接管理器建立文件传输通道
4. SFTP/SCP 引擎执行文件传输操作
5. 传输过程中，SFTP/SCP 引擎通过 IPC 向前端发送进度更新
6. 传输完成后，SFTP/SCP 引擎通过 IPC 向前端发送完成通知
7. 前端更新 UI，显示传输结果

## 6. 数据流设计

### 6.1 连接配置数据流

```
┌───────────────┐     ┌───────────────┐     ┌─────────────────────┐
│ 用户输入       │────▶│ 前端表单验证   │────▶│ IPC 通信            │
└───────────────┘     └───────────────┘     └────────────┬────────┘
                                                          │
┌───────────────┐     ┌───────────────┐     ┌────────────▼────────┐     ┌─────────────────────┐
│ 配置存储       │◀────│ 配置管理模块   │◀────│ SSH 连接管理器      │◀────│ 后端验证与处理      │
└───────────────┘     └───────────────┘     └─────────────────────┘     └─────────────────────┘
```

1. 用户在前端输入连接配置信息
2. 前端表单验证确保输入数据格式正确
3. 前端通过 IPC 将配置数据发送到后端
4. 后端 SSH 连接管理器验证配置数据的有效性
5. 验证通过后，配置管理模块将配置数据存储到配置文件中

### 6.2 终端输入输出数据流

```
┌───────────────┐     ┌───────────────┐     ┌─────────────────────┐
│ 用户键盘输入   │────▶│ 终端 UI 组件   │────▶│ IPC 通信            │
└───────────────┘     └───────────────┘     └────────────┬────────┘
                                                          │
┌───────────────┐     ┌───────────────┐     ┌────────────▼────────┐     ┌─────────────────────┐
│ 终端显示更新   │◀────│ 终端 UI 组件   │◀────│ IPC 通信            │◀────│ 终端仿真引擎        │
└───────────────┘     └───────────────┘     └────────────┬────────┘     └────────┬───────────┘
                                                          │                        │
┌───────────────┐     ┌───────────────┐     ┌────────────▼────────┐     ┌────────▼───────────┐
│ 远程服务器响应 │◀────│ SSH 连接      │◀────│ SSH 连接管理器      │◀────│ 终端输入处理        │
└───────────────┘     └───────────────┘     └─────────────────────┘     └─────────────────────┘
```

1. 用户通过键盘在终端 UI 输入命令
2. 终端 UI 组件捕获输入事件，通过 IPC 发送到后端
3. 终端仿真引擎处理输入数据，转换为适合 SSH 传输的格式
4. SSH 连接管理器将处理后的输入数据发送到远程服务器
5. 远程服务器执行命令，返回输出结果
6. SSH 连接管理器接收输出结果，传递给终端仿真引擎
7. 终端仿真引擎处理输出结果，包括解析 ANSI 转义序列
8. 处理后的输出数据通过 IPC 发送到前端
9. 终端 UI 组件更新显示，呈现输出结果

```
┌─────┴─────┐                                  ┌───────┴───────┐
│ 本地文件  │                                  │ 远程文件      │
└───────────┘                                  └───────────────┘
```

1. 用户在文件传输 UI 选择要传输的文件或目录
2. 文件传输 UI 通过 IPC 将传输请求发送到后端
3. SFTP/SCP 引擎初始化传输任务，获取 SSH 连接
4. SFTP/SCP 引擎执行文件传输操作：
   - 上传：从本地文件系统读取文件，通过 SSH 连接发送到远程服务器
   - 下载：从远程服务器读取文件，通过 SSH 连接保存到本地文件系统
5. 传输过程中，SFTP/SCP 引擎通过 IPC 向前端发送进度更新
6. 传输完成后，SFTP/SCP 引擎更新文件系统状态

### 6.4 日志数据流程

```
┌───────────────┐     ┌───────────────┐     ┌─────────────────────┐
│ 系统事件      │────▶│ 日志记录模块   │────▶│ 日志格式化         │
└───────────────┘     └───────────────┘     └────────────┬────────┘
                                                          │
┌───────────────┐     ┌───────────────┐     ┌────────────▼────────┐
│ 日志查询 UI   │◀────│ IPC 通信       │◀────│ 日志存储           │
└───────────────┘     └───────────────┘     └─────────────────────┘
      ▲                                                ▲
      │                                                │
┌─────┴─────┐                                  ┌───────┴───────┐
│ 用户日志查询│                                  │ 日志文件/数据库 │
└───────────┘                                  └───────────────┘
```

1. 系统各模块产生事件和操作
2. 日志记录模块捕获这些事件，根据日志级别进行过滤
3. 日志记录模块格式化日志数据
4. 格式化后的日志数据存储到日志文件或数据库中
5. 用户通过日志查询 UI 发送查询请求
6. 前端通过 IPC 调用日志记录模块的查询方法
7. 日志记录模块从存储中检索符合条件的日志数据
8. 检索结果通过 IPC 返回给前端，前端更新 UI 显示日志

## 8. 安全架构

### 8.1 凭证安全存储

1. **加密存储机制**：
   - 密码和敏感凭证信息使用平台原生安全存储进行加密存储
   - 支持 Windows 凭证管理器、macOS Keychain 和 Linux Secret Service API
   - 避免在内存中明文存储敏感信息

2. **访问控制**：
   - 凭证访问需要应用级授权
   - 敏感操作需要用户确认
   - 支持可选的主密码保护

3. **安全擦除**：
   - 敏感数据使用后立即从内存中安全擦除
   - 采用符合行业标准的擦除算法

### 8.2 SSH 连接安全

1. **协议支持**：
   - 仅支持 SSH-2 协议
   - 禁用所有不安全的 SSH 协议版本和算法

2. **主机密钥验证**：
   - 严格的主机密钥验证策略
   - 首次连接时提示用户确认主机密钥
   - 存储已验证的主机密钥，防止中间人攻击

3. **加密算法**：
   - 使用强加密算法，如 AES-256-GCM、ChaCha20-Poly1305
   - 禁用弱加密算法，如 3DES、RC4

4. **认证安全**：
   - 支持多种安全认证方式
   - 密码认证时使用安全的密码输入机制
   - 私钥认证时支持密钥加密

### 8.3 IPC 通信安全

1. **通信加密**：
   - Tauri 2 默认提供 IPC 通信加密
   - 确保前端和后端之间的通信安全

2. **API 授权**：
   - 严格的 API 访问控制
   - 防止未授权的 API 调用

3. **输入验证**：
   - 所有 IPC 输入都经过严格验证
   - 防止注入攻击和恶意输入

### 8.4 安全编码实践

1. **最小权限原则**：
   - 应用仅请求必要的系统权限
   - 各模块仅拥有完成其功能所需的最小权限

2. **错误处理**：
   - 避免在错误消息中泄露敏感信息
   - 统一的错误处理机制

3. **日志安全**：
   - 日志中不包含敏感信息
   - 日志文件的访问权限控制

4. **依赖安全**：
   - 定期更新依赖库
   - 对依赖库进行安全审计
   - 使用经过验证的安全库

## 9. 性能优化策略

### 9.1 非阻塞 I/O 设计

1. **异步编程模型**：
   - 使用 Rust 的异步/await 语法进行非阻塞 I/O 操作
   - 避免阻塞主线程，提高应用响应性

2. **事件驱动架构**：
   - 采用事件驱动的设计模式
   - 高效处理大量并发连接

### 9.2 高效 IPC 通信

1. **批量数据传输**：
   - 终端数据采用批量传输方式，减少 IPC 调用次数
   - 优化数据序列化/反序列化性能

2. **事件压缩**：
   - 高频事件采用压缩算法，减少数据传输量
   - 合并相似事件，降低处理开销

### 9.3 终端缓冲区优化

1. **高效的终端渲染**：
   - 使用增量渲染技术，只更新变化的部分
   - 优化终端缓冲区管理，减少内存使用

2. **滚动缓冲区优化**：
   - 实现高效的滚动缓冲区算法
   - 支持配置缓冲区大小，适应不同需求

### 9.4 内存管理优化

1. **内存池**：
   - 使用内存池管理频繁分配/释放的内存
   - 减少内存碎片

2. **按需加载**：
   - 大型资源采用按需加载方式
   - 及时释放不再使用的资源

### 9.5 并发会话处理

1. **连接池管理**：
   - 实现高效的连接池，复用 SSH 连接
   - 支持连接池大小配置

2. **并行处理**：
   - 文件传输采用并行处理方式，提高传输速度
   - 支持配置并行传输数量

## 13. 附录

### 13.1 架构决策记录（ADR）

#### ADR-001：选择 Tauri 2 作为框架

**决策**：选择 Tauri 2 作为跨平台桌面应用框架

**背景**：需要一个跨平台的桌面应用框架，结合 Rust 的安全性和性能优势

**选项**：

- Electron：成熟的跨平台框架，但资源占用高
- Qt：功能强大，但学习曲线陡峭
- Tauri：新兴的跨平台框架，结合 Rust 后端和 Web 前端，资源占用低

**理由**：

- 资源占用低，启动速度快
- Rust 提供的安全性和性能优势
- 现代化的 Web 前端技术栈
- 活跃的社区和持续的发展

**影响**：

- 开发团队需要掌握 Rust 和 Web 前端技术
- 需要适应 Tauri 的主进程-渲染进程模型

#### ADR-002：选择 russh 作为 SSH 库

**决策**：选择 russh 作为 SSH 协议实现库

**背景**：需要一个 Rust 实现的 SSH 库，用于处理 SSH 连接和通信

**选项**：

- russh：纯 Rust 实现的 SSH 库，性能高，安全性好
- ssh2-rs：基于 libssh2 的 Rust 绑定，成熟稳定

**理由**：

- 纯 Rust 实现，安全性更高，避免依赖 C 库
- 良好的异步支持，适合非阻塞 I/O 设计
- 活跃的开发和维护
- 支持最新的 SSH 协议特性

**影响**：

- 需要适应 russh 的 API 设计
- 可能需要自行实现一些高级 SSH 功能
