# 缺失功能文档 (Gap Analysis)

## 文档概述

本文档基于对 `star_-shuttle` 远程终端管理工具的代码分析（版本：2026年1月），对比项目需求文档（`feature_requirements.md` 和 `functional_specification.md`），系统性地识别并记录了当前实现中缺失或未完全实现的功能模块。本文档旨在为后续开发工作提供明确的技术债务清单和优先级指导。

## 分析范围

- **前端**：Svelte 组件 (`src/components/`)
- **后端**：Rust 模块 (`src-tauri/src/modules/`)
- **核心服务**：终端、连接、SFTP、存储等 (`src/lib/`)
- **需求基准**：`feature_requirements.md` 和 `functional_specification.md`

## 详细分析

### 1. 端口转发与隧道（部分实现但未启用）

#### 代码现状
- 文件：`src-tauri/src/modules/connection/ssh_impl.rs`
- 函数：`setup_port_forwarding()` 被大量注释（约第 380-450 行）
- UI 组件：`ConnectionModal.svelte` 包含端口转发配置项

#### 具体缺失
1. **动态端口转发（SOCKS 代理）**
   - SSH 命令：`ssh -D 1080 user@host`
   - 用途：创建本地 SOCKS5 代理，通过 SSH 隧道转发所有 TCP 流量
   - 现状：完全未实现

2. **远程端口转发（-R）**
   - SSH 命令：`ssh -R 8080:localhost:80 user@host`
   - 用途：将远程服务器的端口转发到本地服务
   - 现状：UI 有配置，但后端未激活

3. **本地端口转发（-L）**
   - SSH 命令：`ssh -L 3306:localhost:3306 user@host`
   - 用途：将本地端口转发到远程服务
   - 现状：UI 有配置，但后端未激活

#### 影响
- 运维人员无法通过 SSH 隧道访问内网服务（数据库、Web 管理界面等）
- 缺乏网络调试和穿透防火墙的核心能力

### 2. 文件传输（功能不完整）

#### 双面板缺失
- **现状**：只有远程文件浏览器 (`FileExplorer.svelte`)，**无本地面板**
- **影响**：无法实现本地⇄远程的拖拽互传，用户需依赖操作系统文件管理器

#### 无 SCP 备用
- **现状**：仅实现 SFTP 协议（`sftpService.ts` 和 Rust SFTP 模块）
- **影响**：当目标服务器不支持 SFTP（仅支持 SCP）时，文件传输完全失败

#### 无队列管理
- **现状**：每次传输独立启动，无统一管理界面
- **缺失功能**：
  - 传输任务列表（进行中、等待中、已完成）
  - 暂停/恢复单个传输
  - 调整传输顺序
  - 实时传输速度显示

#### 无断点续传
- **现状**：大文件传输中断后需重新开始
- **影响**：网络不稳定环境下的用户体验极差

### 3. 身份验证（高级特性缺失）

#### MFA/2FA 交互优化
- **现状**：基础 `Keyboard-Interactive` 认证流程
- **缺失**：
  - 清晰的验证码输入弹窗
  - Google Authenticator/Duo 等 MFA 提供商的专用界面
  - 验证码自动填充建议

#### 证书认证不完整
- **代码位置**：`ssh_impl.rs` 第 405-408 行
```rust
// TODO: Implement certificate authentication
return Err("Certificate authentication not supported yet".into());
```
- **影响**：企业环境中基于 SSH 证书的自动化认证无法使用

### 4. 终端增强（效率工具缺失）

#### 多路复用/广播输入
- **场景**：同时管理 10 台同构服务器，需执行相同命令
- **现状**：必须逐个终端手动输入
- **建议实现**：
  - 终端标签多选功能
  - 广播输入模式开关
  - 输入同步状态指示器

#### 快捷命令片段库
- **现状**：无命令保存/管理功能
- **缺失功能**：
  - 预定义命令片段（支持参数占位符 `{host}`, `{date}`）
  - 分类组织（Docker、Kubernetes、系统监控等）
  - 一键发送到当前终端

#### WebGL 渲染加速
- **现状**：使用标准 xterm.js Canvas 渲染器
- **影响**：输出大量文本（`cat large.log`）时 CPU 占用率高，滚动卡顿
- **建议**：集成 `xterm-addon-webgl` 插件

### 5. 监控与审计（企业级功能缺失）

#### 实时流量监控
- **缺失**：无网络 I/O 速率显示（上传/下载 KB/s）
- **建议位置**：终端状态栏或独立监控面板

#### 操作审计日志
- **高危指令**：`rm -rf /`, `reboot`, `shutdown`, `dd`, `mkfs`, `chmod 777` 等
- **建议实现**：
  - 后端 Shell 输出解析（正则匹配）
  - 高危操作警告弹窗（需确认）
  - 审计日志存储（本地加密）
  - 导出报表功能

#### 统计仪表盘
- **缺失指标**：
  - 最常访问主机 Top 10
  - 连接时间分布（按日/周）
  - 文件传输总量统计
  - 终端命令频率分析

### 6. 集成与扩展（生态连接缺失）

#### 本地 Shell 集成
- **现状**：仅支持远程 SSH 终端
- **需求**：在应用内打开本地终端（PowerShell、WSL、Bash、Zsh）
- **价值**：统一本地与远程开发体验

#### 外部编辑器集成
- **场景**：SFTP 中双击文本文件，用 VS Code 打开编辑
- **缺失**：无编辑器关联配置和自动同步机制

#### 云同步
- **现状**：配置存储在本地 SQLite，无跨设备同步
- **建议方案**：
  - GitHub Gist（个人用户）
  - S3/兼容对象存储（企业用户）
  - WebDAV（私有部署）

### 7. 辅助功能（无障碍支持缺失）

#### 屏幕阅读器支持
- **现状**：Svelte 组件缺乏 ARIA 标签
- **影响**：视障用户无法使用 NVDA、VoiceOver 等辅助技术

#### 高对比度模式
- **现状**：仅亮/暗主题，无专门的高对比度主题
- **建议**：WCAG 2.1 AA 合规性要求
